## Overview

A production-grade Extract вҶ’ Transform вҶ’ Load pipeline built for the retail industry. This project extracts raw sales, inventory, product, returns and store data from a PostgreSQL database, validates and cleans it, and produces a processed dataset ready for exploratory data analysis and machine learning.

## Pipeline Classes

### `DataValidator`
Diagnoses data quality issues without modifying anything. Like a doctor вҖ” inspects and reports only.

```python
validator = DataValidator(df)
validator.run()
print(validator.report)
```

### `DataTransformer`
Implements fixes based on the validator's findings. Always works on a copy. Records every change.

```python
transformer = DataTransformer(validator)
transformer.run()
print(transformer.summary)
```

### `ETLPipeline`
Orchestrates the full pipeline with method chaining.

```python
pipeline = ETLPipeline()
pipeline.extract().validate().transform().load()
```

## Data Quality Checks

| Check | Dimension | Action |
|-------|-----------|--------|
| Null values | Completeness | Fill with median / "Unknown" |
| Duplicate rows | Uniqueness | Drop вҖ” keep first |
| Negative revenue/cost | Validity | Set to null |
| Discount > 100% | Validity | Flag as error |
| Mixed casing | Consistency | Standardise to title case |
| Duplicate columns (.1) | Schema | Flag for SQL query fix |
| stock_status = 'NO DATA' | Completeness | Flag missing inventory |
